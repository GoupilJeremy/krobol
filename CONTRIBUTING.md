# **🤝 Contribuer à Krobol**

Krobol est un projet **open source** et **communautaire** ! Toute contribution est la bienvenue, que ce soit pour corriger un bug, ajouter une nouvelle fonctionnalité, améliorer la documentation ou traduire le projet.

---

## **📌 Comment Contribuer ?**

### **1. Signaler un Bug ou Proposer une Amélioration**
- **Ouvrez une issue** sur [GitHub Issues](https://github.com/GoupilJeremy/krobol/issues).
- Utilisez l’un des **templates** suivants :
  - **Bug Report** : Pour signaler un dysfonctionnement.
  - **Feature Request** : Pour proposer une nouvelle fonctionnalité.
  - **Enhancement** : Pour suggérer une amélioration.

### **2. Contribuer du Code**
1. **Forkez le dépôt** :
   ```bash
   git clone https://github.com/GoupilJeremy/krobol.git
   cd krobol
   git checkout -b feature-votre-nouvelle-fonctionnalite
   ```
2. **Implémentez vos changements** dans le code source.
3. **Ajoutez des tests** pour valider vos modifications.
4. **Commitez vos changements** :
   ```bash
   git commit -m "[Feat] Ajout de la fonctionnalité X"
   ```
5. **Poussez votre branche** :
   ```bash
   git push origin feature-votre-nouvelle-fonctionnalite
   ```
6. **Ouvrez une Pull Request (PR)** sur GitHub.

### **3. Améliorer la Documentation**
- Modifiez les fichiers dans le dossier `docs/`.
- Ajoutez des **guides**, **tutoriels** ou **exemples d’utilisation**.
- Traduisez la documentation dans d’autres langues.

### **4. Rejoindre les Discussions**
- Participez aux discussions sur [GitHub Discussions](https://github.com/GoupilJeremy/krobol/discussions).
- Posez des questions, partagez vos idées ou aidez d’autres contributeurs.

---

## **📝 Bonnes Pratiques**

### **Pour les Issues**
- **Soyez précis** : Décrivez clairement le problème ou la proposition.
- **Ajoutez des détails** : Code, captures d’écran, logs ou étapes pour reproduire le bug.
- **Utilisez les labels** : `bug`, `enhancement`, `documentation`, etc.

### **Pour les Pull Requests**
- **Respectez le style de code** du projet (utilisez ESLint/Prettier si disponible).
- **Documentez votre code** : Ajoutez des commentaires et mettez à jour la documentation.
- **Testez vos modifications** : Vérifiez que tous les tests passent.
- **Soyez concis** : Une PR doit résoudre un problème ou ajouter une fonctionnalité claire.
